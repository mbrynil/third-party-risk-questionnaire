Implement Vendor Dashboard V1 (vendor directory + vendor profile) with a clean, professional Tailwind UI. Keep existing “Quick Create questionnaire” flow on the front page.

1) Data model

Create a Vendor entity/table:
	•	id
	•	company_id / owner_id (whatever you use to scope data to Company A)
	•	name (required)
	•	primary_contact_name (nullable)
	•	primary_contact_email (nullable)
	•	notes (nullable)
	•	status (enum/string: ACTIVE default, ARCHIVED)
	•	created_at, updated_at

Update Questionnaire to link to vendors:
	•	add vendor_id (nullable for backwards compatibility)
	•	keep existing vendor fields if you have them, but new flows should use vendor_id

2) Keep existing Quick Create flow (front page)

Do NOT remove the current flow:
	•	Front page can still create a questionnaire and accept vendor info inline.

But under the hood:
	•	When a user quick-creates a questionnaire and enters vendor name/contact:
	•	If a vendor with same name (case-insensitive) exists for this company, link to it
	•	Else create a new Vendor record and link the questionnaire to it via vendor_id

6) Create New Assessment from Vendor Profile (Company A)

From Vendor Profile, clicking + New Assessment should open a modal or page that lets the user choose:

Option A: “From Template”
	•	pick a template (templates scoped to this company)
	•	create questionnaire from template
	•	link it to this vendor_id
	•	redirect to the questionnaire builder/config page (NOT vendor link page)

Option B: “From Question Bank”
	•	create a new empty questionnaire linked to vendor_id
	•	redirect to questionnaire builder/config page

After builder/config, user proceeds to “Get Vendor Link” as already implemented.
